# パラメータリファレンス

全シミュレーションパラメータ(`SimParams`、[src/sim/mod.rs](../src/sim/mod.rs))と顔料個性の意味・既定値・UI 範囲の一覧。**定義の正典はコード側のフィールドコメント**であり、本書は一覧性のためのリファレンス。プリセットは `assets/presets/*.json`(serde default で古い JSON も読める)。

最終更新: 2026-07-09(**ブランチ refactor/capillary-water の試作**: 水の毛細管拡散([note/07](note/07-capillary-water.md))で置き馴染みパッチ群を一次原理化。`paint_spread` を削除し、塗るツールのノブは `paint_pickup`/`brush_charge` の2本に。§4 末尾の定数表参照)

## 1. ブラシ

| パラメータ | 既定値 | UI 範囲 | 意味 |
|---|---|---|---|
| `brush_radius` | 16.0 | 1〜64 | ブラシ半径(テクセル単位) |
| `brush_water` | 0.5 | 0〜2 | 1 splat あたりに置く水量 |
| `brush_velocity` | 0.5 | 0〜2 | splat 時にストローク方向へ与える初速 |
| `brush_pigment` | 0.11 | 0〜1 | 1 splat あたりに置く顔料量(0 = 水だけのブラシ) |
| `brush_channel` | 0 | 色見本ボタン | 注入する顔料スロット(0〜3 = パレットの4スロットと対応。§9) |
| `tool` | 0 | ツールボタン | 0=描画 / 1=リフト(削り) / 2=消去 / 3=水筆 / 4=ならし(splat.wgsl が分岐) |
| `lift_strength` | 0.3 | 0〜1 | リフト(tool=1)の基準強度。実効値は顔料の ω と紙ハイト(凸部ほど剥がれる)で変調 |
| `water_lift` | 0.4 | 0〜1 | 水筆(tool=3)の均し強度: ブラシ下の顔料を近傍平均(半径2固定)へ寄せる緩和率。質量保存の箱ぼかしなので濃くならない |
| `smear_rate` | 0.35 | 0〜1 | ならし(tool=4)の均し強度: 総顔料をブラシスケールの近傍(`clamp(半径/4, 2, 10)`)へ均す緩和率。濃い山を周囲へ均一に伸ばす |
| `paint_pickup` | 0.2 | 0〜1 | 色の溶かし戻し: 塗る筆が触れた沈着顔料を浮遊層へ戻す割合(coverage×筆圧×(1−ω) で変調。リフト tool=1 の弱い版)。筆の色と下の色が同じ浮遊層で混ざり、置いた境界が馴染む。feed splat 経由(筆の含み)では毎フレーム適用になるため 0.1 倍に減衰される。0=無効 |
| `brush_charge` | 2.0 | 0〜5 秒 | 筆の含み: 塗る筆を置いたまま動かさなくても、この秒数だけ色水が流れ出続ける(CPU が毎フレーム feed splat を積む=目標水位への水補充(max)+`brush_pigment`×`CHARGE_PIGMENT`(=0.15、splat.wgsl の定数)ぶんの顔料。`Splat.feed` フラグで分岐)。筆を下ろすたびに満たされる。動いているフレームは通常注入があるので消費しない。0=無効(従来どおり) |

「置いた色が外へ広がる」挙動はパラメータではなく**水の毛細管拡散**([note/07](note/07-capillary-water.md)、§4 の定数 `WATER_DIFFUSE`/`WATER_GAMMA`)が担う。広がる距離は置く水の量(`brush_water`)と含み(`brush_charge`)で決まる。

| パラメータ | 既定値 | UI 範囲 | 意味 |
|---|---|---|---|
| `pressure_radius` | 0.6 | 0〜1 | 筆圧→半径の効き(0=筆圧無効 / 1=完全比例) |
| `pressure_water` | 0.3 | 0〜1 | 筆圧→水量の効き |
| `pressure_pigment` | 0.7 | 0〜1 | 筆圧→顔料量の効き |
| `pressure_gamma` | 1.0 | 0.25〜4 | 応答カーブ γ(筆圧^γ)。1=線形、>1 で軽いタッチがより細く弱くなる |

## 3. 水の流体(velocity / relax / advect)

| パラメータ | 既定値 | UI 範囲 | 意味 |
|---|---|---|---|
| `dt` | 0.5 | 0.05〜1 | シミュレーションの時間刻み |
| `accel` | 1.0 | 0〜4 | 移流強度: 水面勾配 → 加速度の係数。水面 = 水深 + `paper_amp`×紙ハイト |
| `damping` | 0.05 | 0〜0.5 | 速度減衰(粘性の代用。1ステップあたりの減衰率) |

| `relax_iters` | 16 | 1〜50 | 発散緩和の反復回数/ステップ(安全弁の上限 64) |

| `wet_expand` | 0.0 | 0〜0.5 | にじみ拡張率: 乾いたセルのマスクが濡れた隣の水量に比例して育つ速さ(0.5 超で濡れに昇格)。0 = 筆が通った領域に固定 |

## 4. 顔料の輸送(diffuse / transfer)

| パラメータ | 既定値 | UI 範囲 | 意味 |
|---|---|---|---|
| `pigment_diffuse` | 0.4 | 0〜1 | 顔料拡散率(1反復あたり。安定条件のため WGSL 側で 0.2 に制限。既定 0.4×dt0.5=上限に張り付き=水たっぷりのセル間は最速で混ざる) |
| `diffuse_iters` | 12 | 0〜32 | 拡散の反復回数/ステップ。実効的なにじみ速度 = 拡散率 × 反復回数。速いにじみはこちらで稼ぐ |
| `deposit_rate` | 0.05 | 0〜0.5 | 吸着率: 浮遊顔料が紙へ沈着する速さ(水が少ないほど強い)。顔料の ρ と粒状ゲートで変調 |
| `lift_rate` | 0.1 | 0〜0.5 | 脱着率: 沈着顔料が水に浮き上がる速さ(水が多いほど強い)。顔料の (1−ω) で変調。0.02→0.1(note/07): 毛細管モデルでは既存色が新しい塗りへ戻る経路が再浮遊→拡散だけなので速めが既定 |
| `evap_rate` | 0.005 | 0〜0.05 | 蒸発率: 濡れ領域の水が 1 ステップに減る量 |

transfer.wgsl の実効式(per-channel、M3):

```
吸着 = deposit_rate × ρ × dt × deposit_weight(水量) × 粒状ゲート(paper_gran×γ)
       deposit_weight(w) = mix(2.0, DEPOSIT_WET_FLOOR, smoothstep(0, DEPOSIT_DRY_POINT, w))
脱着 = lift_rate × dt × 水量 × (1−ω)
```

### 調整済みの WGSL 定数(SimParams ではない)

作画中に変えない「シミュの性格」は SimParams ではなく各シェーダーの `const` に置く(note/06 §8 で定数化、[note/07](note/07-capillary-water.md) で毛細管化)。再調整はホットリロード(H1)でシェーダーを直接編集する(SimParams のレイアウトが変わらないので再起動不要):

| 定数 | 値 | 場所 | 意味 |
|---|---|---|---|
| `FLOW_MIN` | 0.3 | [velocity.wgsl](../assets/shaders/velocity.wgsl) | 薄い水膜の流れ止め(潤滑近似): この水量以下で速度が消えはじめる。乾きかけの紙目駆動の流れが浮遊顔料を谷へ寄せてざらつくのを防ぐ。吸着の立ち上がり(DEPOSIT_DRY_POINT)よりわずかに下 |
| `WATER_DIFFUSE` | 0.15 | [diffuse.wgsl](../assets/shaders/diffuse.wgsl) | 水の毛細管拡散の速さ(per-iter 上限 0.2、×diffuse_iters)。置いた水が濡れた紙を伝って広がる(note/07。旧・置き馴染み+広がる勢いの置き換え) |
| `WATER_GAMMA` | 2.0 | [diffuse.wgsl](../assets/shaders/diffuse.wgsl) | 水の毛細管の水依存カーブ: 流束の重み = max(両セル水量)^γ。max なのは乾きかけへ吸い込まれる向きを殺さないため |
| `DIFFUSE_GAMMA` | 4.0 | [diffuse.wgsl](../assets/shaders/diffuse.wgsl) | 顔料の混ざりの水依存 γ: 拡散流束の重み = (水量平均)^γ。水たっぷり=自由に混ざる/引くと止まる |
| `CHARGE_PIGMENT` | 0.15 | [splat.wgsl](../assets/shaders/splat.wgsl) | 筆の含みの feed splat 1フレームが注ぐ顔料の通常比。0 で水だけ |
| `DEPOSIT_WET_FLOOR` | 0.1 | [transfer.wgsl](../assets/shaders/transfer.wgsl) | 水膜がある間(w ≥ DRY_POINT)の吸着倍率。0 にすると濡れている間は全く沈まない(旧 (2−w)×(1−0.7w) の2重カーブを deposit_weight 1本に統合。旧「水持ち」を含む) |
| `DEPOSIT_DRY_POINT` | 0.35 | [transfer.wgsl](../assets/shaders/transfer.wgsl) | 吸着が立ち上がる水量しきい値(=水膜が切れる点)。上げるほど早く沈み始め、下げるほど流れている間は浮遊し続ける |

## 5. 紙(paper.rs / 3箇所適用)

紙ハイトは起動時に CPU 生成する静的テクスチャ(値ノイズ4成分: 低周波の沈殿ムラ+中周波の紙目+微細なざらつき+横方向の繊維ストリーク)。

**シミュ側の紙の質感再現は既定オフ(全て 0。2026-07-09 のユーザー判断、capillary ブランチ)**。機構は残っているのでスライダーで戻せる。線画の鉛筆粒状感(`pencil_gran`、§10)と、リフトの紙凸部変調(顔料 γ 依存、splat.wgsl)は対象外でそのまま。

| パラメータ | 既定値 | UI 範囲 | 意味 |
|---|---|---|---|
| `paper_amp` | 0.0 | 0〜1 | 紙ハイトの振幅: 水面勾配に足す凹凸の強さ(水が谷へ流れる → ストリーク) |
| `paper_gran` | 0.0 | 0〜1 | 粒状化: 紙の凹部で吸着が強まる度合い(顔料の γ と乗算。0=紙目の影響なし)。**0 だと顔料個性 γ(粒状感)はほぼ効かなくなる** |
| `paper_wet` | 0.0 | 0〜1 | にじみ拡張の紙目変調: 濡れ前線が谷を選んで進む度合い(縁が不規則に。0=一様) |

## 7. 乾燥(M2、bake / rewet)

| パラメータ | 既定値 | UI 範囲 | 意味 |
|---|---|---|---|
| `dry_shift` | 1.0 | 0〜1.5 | 焼き込み時に顔料濃度へ掛ける係数。**既定 1.0 = 乾いても濃度を保つ**(アナログ調に薄くしたいときだけ <1 に下げる) |
| `dry_gran` | 0.0 | 0〜1 | 焼き込み時の粒状感ゲート: 紙の凹部で濃く/凸部で薄く定着(描画中の `paper_gran` とは独立) |
| `dry_edge` | 0.4 | 0〜2 | 焼き込み時のエッジダークニング: 顔料被覆率の縁バンド(ぼかし残差)で濃度を増す強さ。効きは弱め(既知、status.md 参照) |

## 9. 顔料個性 ρ/ω/γ(M3、[crates/pigment/src/lib.rs](../crates/pigment/src/lib.rs))

SimParams とは別に、顔料ごとの物性3値を uniform(binding 9)で渡す。**M5 でランタイム編集可能**([`Palette`]、パレットパネルで色・ρ/ω/γ を編集 → `set_palette` が physics/latents バッファへ即時反映)。ρ/ω/γ は湿シミュ専用=全レイヤー共通で、編集は過去の乾燥レイヤーに遡らない。色(latent)は「乾かす」時にレイヤーへ記録するため乾燥済みは変色しない(M5c)。**per-顔料の密度 ρ(`density`、沈着の速さ)と、表示の被覆率カーブ `pigment_density`(§8)は別概念**(M5a で用語を分離)。

| 物性 | 範囲(UI) | 意味 |
|---|---|---|
| 密度 ρ (`density`) | 0.1〜2.0 | 沈着(吸着)の速さ。重い顔料ほど早く沈む(transfer の吸着を倍率変調) |
| ステイニング ω (`staining`) | 0〜1 | 剥がれにくさ。高いほど脱着・リフトが (1−ω) で抑えられ紙に色が残る(ステイン床) |
| 粒状感 γ (`granulation`) | 0〜1 | 紙ハイトへの反応。高いほど凹部(谷)に沈着し凸部で剥がれる = 粒状化 |

既定パレット(mixbox 推奨顔料色、対比を付けた代表値。UI で編集可):

| スロット | 顔料 | sRGB | ρ | ω | γ | 性格 |
|---|---|---|---|---|---|---|
| 0 | ハンザイエロー | 252,211,0 | 1.0 | 0.5 | 0.1 | 半透明・中ステイニング・ほぼ非粒状 |
| 1 | フタロブルー | 13,27,68 | 0.8 | 0.9 | 0.05 | 微細粒子。強ステイニング(剥がれない)・非粒状・軽い |
| 2 | キナクリドンマゼンタ | 128,2,46 | 0.9 | 0.85 | 0.05 | 透明・強ステイニング・非粒状 |
| 3 | バーントシェンナ | 123,72,0 | 1.3 | 0.2 | 0.8 | アース系。重く早く沈着・低ステイニング(よく剥がれる)・強粒状(紙目に溜まる) |

この4色は色単体ライブラリ(`assets/pigments/*.json`、M5f)にも同じ値で同梱してある(1色=1ファイル、`Pigment` の serde そのまま。スキーマは本節の3値+名前+sRGB で不変)。

## 10. 線画(M4.5、linesplat.wgsl / display.wgsl)

鉛筆・ペン・ハイライトのラスタツール。流体シミュを通らず、`linesplat.wgsl` が対象の線画テクスチャ(r32float、鉛筆用/ペン用/ハイライト用の各1枚)へインク濃度(ハイライトは不透明度)を直接蓄積する。半径・筆圧の効きは「1. ブラシ」「2. 筆圧」の値を共用。

| パラメータ | 既定値 | UI 範囲 | 意味 |
|---|---|---|---|
| `line_mode` | 0 | ツールボタン | ラスタ線画の種別: 0=鉛筆(グレー粒状線・筆圧→濃さ)/ 1=ペン(濃色スムーズ線・筆圧→太さ)/ 2=ハイライト(不透明白・筆圧→不透明度、M4.5c)。描画先テクスチャは bind group で選ぶ |
| `line_eraser` | 0 | チェックボックス | ラスタ消しゴム: 0=描画(蓄積)/ 1=減算 |
| `pencil_radius` | 8.0 | 1〜64 px | 鉛筆の半径。水ブラシ(`brush_radius`)とは独立。筆圧で締められる |
| `pencil_strength` | 0.7 | 0〜1 | 鉛筆のインク濃度の基準(筆圧で変調) |
| `pencil_gran` | 0.5 | 0〜1 | 鉛筆の粒状感: 紙ハイトでインク濃度を変調する度合い(0=一様 / 1=山に強く乗る) |
| `pen_radius` | 4.0 | 1〜64 px | ペンの半径。水ブラシ(`brush_radius`)とは独立。筆圧→太さ |
| `pen_strength` | 0.9 | 0〜1 | ペンのインク濃度の基準(満量で掛かる) |
| `line_block` | 0.0 | 0〜1 | **清書ペン線の透水率(M4.5b)**。透水率 `perm = 1 − line_block×ペン濃度` を ①拡散の隣接流束(diffuse)②にじみ拡張の蓄積(velocity)③速度場(velocity)に掛ける。上げるほどペンで囲った領域を塗っても水が線を越えない。**ブラシの直接スプラットには掛けない**(線を跨ぐ筆使いなら明示的に越えられる)。0=境界なし |
| `highlight_radius` | 6.0 | 1〜64 px | ハイライト(不透明白)の半径(M4.5c)。筆圧で締められる |
| `highlight_strength` | 0.85 | 0〜1 | ハイライトの不透明度の基準(筆圧で変調、M4.5c) |
| `show_pencil` | 1 | チェックボックス | 下書き(鉛筆)レイヤーの表示(display 合成時に参照) |
| `show_pen` | 1 | チェックボックス | 清書(ペン)レイヤーの表示 |
| `show_highlight` | 1 | チェックボックス | ハイライト(白)レイヤーの表示(M4.5c) |

蓄積は「目標濃度への max」(1フレーム内で密にサンプルしても濃くなりすぎず一定線濃度に収束)。消しゴムのみ減算。表示合成は色(紙→乾燥→湿)の上に鉛筆(グレー)→ペン(濃色)→ハイライト(白)の順でアルファ合成(アルファ = インク濃度/不透明度)。

**多段 Undo/Redo(M4.5d)**: 線画は流体を通らないので決定論的に再ラスタライズできる。生ポインタ点+描画時の実効 SimParams をストローク単位で [src/app/linehist.rs](../src/app/linehist.rs) に保持し、Undo = 対象テクスチャをクリアして残りを引き直す / Redo = 取り消し分を再適用(Ctrl+Z / Ctrl+Shift+Z、Redo は Ctrl+Y も)。保存済みパラメータで引き直すため、スライダーを変えても過去の線の形は変わらない。湿レイヤー(水彩)は対象外(M6 の 1 段 undo)。

## 10.5 アクティブタイル(M6)

| パラメータ | 既定 | UI 範囲 | 意味 |
|---|---|---|---|
| `active_tiles` | 1 | チェックボックス | **アクティブタイル最適化(M6)**。0=全面計算 / 1=濡れタイル+ブラシ近傍のタイルだけシミュレーションを走らせる。シミュコストを「紙の広さ」比例から「濡れ面積」比例へ。0 で従来どおり全面計算に戻せる(A/B・不具合時の退避)。診断表示(H4)の「アクティブタイル」モードで計算中の範囲を目視できる |

`tilescan.wgsl` がタイル(16×16 テクセル)ごとに濡れマスク・水・顔料・ブラシ位置を走査して生の有効フラグを作り、`tiledilate.wgsl` が1タイル分ふくらませて `active` を確定する(にじみ前線の余裕)。各シミュパス(splat/velocity/relax/flowout/advect/diffuse/transfer)は共通レイアウトの binding 11 で `active` を読み、非アクティブなタイルは3テクスチャを素通し(ping-pong 一貫性)して計算を省く。タイルサイズ・タイル数の WGSL 定数(`TILE_SIZE`/`TILES_PER_SIDE`)はキャンバスサイズ依存(M8)のため、[src/gpu/mod.rs](../src/gpu/mod.rs) の `shader_prelude` が生成して各シェーダーの先頭に連結する(Rust 側定義が唯一の正典)。**view(パン/ズーム)と同じく最適化トグルは SimParams だが、これは表示でなく計算範囲を絞るノブ**。
