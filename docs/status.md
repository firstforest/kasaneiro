**KM の設計判断**: 層「内」の混色は M1c の mixbox のまま(RGB 3ch の K,S で混ぜると黄+青が濁り、mixbox に移した意味が消える)。**KM は層「間」の光学スタックだけに使う**。各層の反射率 R・透過率 T は、その層を白地・黒地に置いた mixbox 発色 R_w, R_b から導く(単層を白/黒地に置いた合成式を逆に解くと `R=R_b`, `T²=(R_w−R_b)(1−R_b)` と閉じた形になり sinh/cosh 不要 = オーバーフロー対策も不要)。この簡約が Curtis 一般式と一致することを km.rs のテスト(reduction_matches_general)で担保。**「顔料ごとの K,S プリセット」は不採用**とし、代わりに白/黒地発色から層単位で導出する方式にした(mixbox 隔離 = plan §4 を保ち、混色モデルを二重化しないため)。GPU 配管は黒 latent を pigment.rs の latent_uniform に追加([12],[13])しただけ。パラメータ追加: `compose_mode`(既定 1=KM。SimParams 末尾で 16B 境界を担う)。

M2 完了条件「乾かした色の上に別の色を描くと、下の層が動かず、重なりが透けて見える」は目視確認済み。エッジダークニング(dry_edge)は顔料被覆率ベースの縁バンドに直したが効きが弱いまま — 重要度が低いためこのままとする(強くしたくなったら「縁バンドへ内側から顔料を移す質量保存つき再分配」が次の一手。下記 M2 行参照)。
