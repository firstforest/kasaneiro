## 現在地

**M3 完了 — 削り2ツール・顔料個性(ρ/ω/γ)・KM 合成をすべて目視確認済み。** M3 を2スライスに分けた: ①削り+顔料個性 ②KM 合成。②は [src/km.rs](../src/km.rs)(Kubelka-Munk 純関数の CPU 参照実装+テスト)を新設し、[assets/shaders/display.wgsl](../assets/shaders/display.wgsl) の合成を `compose_mode`(0=multiply / 1=KM)で切替可能にした。cargo test(17件、うち km 4件)・clippy・shader_compile(naga)・実機起動(pipeline 検証通過)に加え、**multiply ⇔ KM を A/B して KM のグレーズ挙動を目視確認済み**。

**M4 着手(進行中)**: 水筆ツール(tool=3)を追加した。水で濡らしつつ、ブラシ下の顔料を近傍平均へ均す(質量保存の箱ぼかし)。均一な塗りに馴染ませ、なでても濃くならない(下表 M4 行参照)。実装・シェーダー検証は通ったが**目視確認はこれから**。

**M5〜M8 を計画に追加(2026-07-05、plan.md §3)**: パレット(顔料の作成・編集・スポイト、M5)→ スケールする土台(アクティブタイル・パン/ズーム・undo 1段、M6)→ 作品保存(M7)→ キャンバス 2048²(M8)。M4 の任意項目のうち保存・undo・アクティブタイル・解像度引き上げを具体化・昇格したもの。同時顔料数の 8ch 化は不採用(レイヤーごとパレット記録 M5c で実質多色化できるため。再検討トリガーは plan.md §4)。

**KM の設計判断**: 層「内」の混色は M1c の mixbox のまま(RGB 3ch の K,S で混ぜると黄+青が濁り、mixbox に移した意味が消える)。**KM は層「間」の光学スタックだけに使う**。各層の反射率 R・透過率 T は、その層を白地・黒地に置いた mixbox 発色 R_w, R_b から導く(単層を白/黒地に置いた合成式を逆に解くと `R=R_b`, `T²=(R_w−R_b)(1−R_b)` と閉じた形になり sinh/cosh 不要 = オーバーフロー対策も不要)。この簡約が Curtis 一般式と一致することを km.rs のテスト(reduction_matches_general)で担保。**「顔料ごとの K,S プリセット」は不採用**とし、代わりに白/黒地発色から層単位で導出する方式にした(mixbox 隔離 = plan §4 を保ち、混色モデルを二重化しないため)。GPU 配管は黒 latent を pigment.rs の latent_uniform に追加([12],[13])しただけ。パラメータ追加: `compose_mode`(既定 1=KM。SimParams 末尾で 16B 境界を担う)。

M2 完了条件「乾かした色の上に別の色を描くと、下の層が動かず、重なりが透けて見える」は目視確認済み。エッジダークニング(dry_edge)は顔料被覆率ベースの縁バンドに直したが効きが弱いまま — 重要度が低いためこのままとする(強くしたくなったら「縁バンドへ内側から顔料を移す質量保存つき再分配」が次の一手。下記 M2 行参照)。
