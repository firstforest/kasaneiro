**分離色: 顔料個性に「粒の細かさ μ」+2色同時に注ぐ筆(2026-07-11、要目視確認)**: ユーザー要望「顔料を2種類含んだ色=分離色を再現したい」への対応。実物の分離色(2顔料混合の絵具)が紙の上で分かれる主因は**粒径の差**(細かい顔料は水に乗って縁まで運ばれ、粗い顔料はその場に残る)だが、従来の顔料拡散は4チャンネル共通の係数で差が出せなかった。**①第4物性「粒の細かさ μ」(`Pigment.mobility`、既定 1.0・UI 0.1〜2.0)**: physics uniform(binding 9)の予備だった w 成分に載せ([crates/pigment/src/lib.rs](../crates/pigment/src/lib.rs))、[diffuse.wgsl](../assets/shaders/diffuse.wgsl) の拡散係数をチャンネル別 `min(pigment_diffuse×dt×μ, 0.2)` に(μ は両セル共通の定数なので流束の対称性=質量保存は不変。安定上限 0.2 もチャンネルごとにクランプ)。uniform レイアウト不変なので **μ の効きはホットリロードで調整可**。既定パレットは フタロ 1.2 / マゼンタ 1.1 / イエロー 1.0 / シェンナ 0.3 の対比(同梱 pigments JSON 4色も更新。ユーザー保存の2色は default 1.0 で従来どおり)。旧 JSON・旧 .kasane は `#[serde(default)]`=1.0 で読める(SimParams と同じ前進互換。pigment crate にテスト追加)。**②分離色ブラシ(`brush_channel2` 既定 3・`brush_mix` 既定 0=単色)**: `_pad_paint` の残り2枠を昇格(SimParams 224B 不変だが**レイアウト変更なので再ビルド・再起動が必要**)。[splat.wgsl](../assets/shaders/splat.wgsl) の描画・feed 両ブランチで総顔料量(`brush_pigment`)を2チャンネルへ比率按分(表示側は変更不要=同セル2チャンネルの mixbox 混色が既に正しい)。分かれは注入でなく既存物理(μ・ρ・ω の差)が起こす: シェンナ(重い・粗い)は中心で早く沈着、フタロ(軽い・細かい・強ステイン)は毛細管で縁へ走る。吸い取り(浮遊のみ取る)が長く浮く細かい色を選択的に取るので分離の強調にも使える。**UI**: 「色をつくる」に4本目のスライダー「粒の細かさ(伸び)」([palette.rs](../src/app/ui/palette.rs))/ 塗るツールに「2色目を混ぜる」スライダー+比率>0 のとき2色目ミニスウォッチ行([tools.rs](../src/app/ui/tools.rs)。「塗る色: A + B(NN%)」表示)。粒状化(paper_gran)は「紙の質感再現は要らない」判断のまま触っていない(分離=色が分かれる、はテクスチャなしで成立)。parameters.md(最終更新・§1・§9・§11=**SimParams の予備パディング消滅、次の追加は _pad 新設から**)・architecture.md(シェーダー一覧・binding 9 のレイアウト)更新済。検証(cargo test --workspace 全34件=μ の範囲/uniform レイアウト/旧 JSON 前進互換のテストを追加 / clippy --workspace 警告0 / naga)。**要目視確認**: たっぷり濡らした紙にフタロブルー+2色目シェンナ(混合比 0.5)で置く → 中心が茶寄り・縁に青のハローで2色に分かれる / μ を両方 1.0 に揃えると分かれが弱まる / 「2色目を混ぜる」0 で従来の単色と同じ / 既定 μ=1.0 の色の描き味が従来と変わらない(注意: シェンナ単色は μ 0.3 になったぶん従来より伸びが鈍い=好みに合わなければ「色をつくる」で μ を上げる)。

**顔料拡散の既定を 0.4→0.2 に(乾燥遅延の随伴調整。2026-07-11、要目視確認)**: 下記の蒸発率変更で濡れている時間が3〜4秒→約1分と15倍になり、顔料が混ざり続ける時間も15倍になったため、ユーザー要望「顔料が広がる速度を少し遅くしたい」に対応。ノブの選択: `diffuse_iters`(12)を減らすと診断の結果**水の毛細管拡散**(diffuse.wgsl の `kw = WATER_DIFFUSE×dt` も反復回数ぶん掛かる)まで遅くなり、調整済みの「置いた色水が遠くへ届く」動きを壊すため、**顔料側の `pigment_diffuse` のみ** 0.4(実効 0.2=安定上限に張り付き)→ 0.2(実効 0.1=半分)に下げた([sim.rs](../crates/paint-core/src/sim.rs))。水依存カーブ(DIFFUSE_GAMMA=4)・反復回数は不変なので「水たっぷりで混ざる/引くと止まる」の性格はそのまま、速度だけ半分。スライダーを 0.4 以上に上げれば旧来の最速へ戻せる。parameters.md §4 更新済。**要目視確認**: 2色を隣接させたときのグラデーションが従来より穏やかに進む / 置いた色水の届く距離(毛細管の広がり)は変わらない / まだ速ければスライダー(開発モード「顔料」)でさらに下げて、決まった値を教えてもらえれば既定化。

**新ツール「吸い取り」= 乾いた筆で浮いた色水を吸う(2026-07-10、要目視確認)**: ユーザー要望「浮いている色を吸い取る機能も欲しい。乾いた筆で絵の具を吸うイメージ」への対応。水彩の thirsty brush(乾筆/スポンジ)に相当する新 WetTool を追加した。**機構**([splat.wgsl](../assets/shaders/splat.wgsl) の tool=4 分岐): `frac = absorb_rate×coverage×筆圧` で**表面の自由水と浮遊顔料を同率で取り除く**(吸った水の顔料濃度=セルの濃度、という物理そのまま。浮遊顔料は水に乗っているだけなので ω=染みつきは関与しない)。沈着顔料には触れない(剥がすのはリフト tool=1)。速度も同率で減衰(水が減るぶん運動量も持ち去られる)。濡れマスクは立てたまま=紙はまだ湿っており、吸った跡へ周囲から毛細管拡散で水が戻る「湿り戻り」も既存物理に任せる。**gpu_id は旧ならしの欠番 4 を再利用**(リリース前につき互換考慮なしの方針。requirements §3 の 5〜7=線画と番号が揃う。tool.rs のテストは from_gpu_id(4)==Some(Absorb) に更新)。既存ツールとの整理: ぼかし筆=水を足して溶かす / 吸い取り=水ごと引き上げる、リフト=沈着→浮遊 / 吸い取り=浮遊→筆、で方向が対になる。**SimParams**: `absorb_rate`(既定 0.5、UI 0〜1)を `_pad_paint` から1枠昇格(224B 不変だが Rust 側フィールド追加+common.wgsl のフィールド名変更なので**再ビルド・再起動が必要**=実行中アプリへの WGSL ホットリロードだけでは整合しない)。UI はツールバー 7→8 ボタン(色4+削り/消す/ぼかし筆/吸い取り)、選択時のみ「吸い取りの強さ」スライダー([tools.rs](../src/app/ui/tools.rs))。ツールバーのボタン・記録(H5)は WetTool::ALL を回すだけなので enum 追加で自動追従。parameters.md(最終更新・§1・§11)・requirements.md §3(ツール表に 4 を追加)・architecture.md(シェーダー一覧)更新済。検証(cargo test --workspace 全32件=gpu_id 往復に Absorb が乗る / clippy --workspace 警告0 / naga コンパイルテスト)。**要目視確認**: たっぷり濡れた塗りを吸い取りでなでる → 水たまりと浮いた色が引いて、なでた跡が明るく抜ける / 沈着済みの色(乾きかけの下地)は残る / 白抜き: 濡れた塗りの中を一筆吸うとハイライトが抜ける / 吸った跡に周囲からじわっと水が戻る / 強さ 0 で何も起きない。

**水の毛細管拡散でアルゴリズムを一次原理化(2026-07-09、目視調整4ラウンドを経て main へマージ=採用済み)**: 「そもそものアルゴリズムは無駄に複雑になってないか」「パラメータが増えると今後の拡張が難しくなる」というユーザー指摘への対応。置き馴染み(水のテレポート)と広がる勢い(relax と綱引きする放射流)は、どちらも**毛細管現象(水自身が濡れた紙を伝って広がる力)がモデルにない**ことへの現象論パッチだったので、[diffuse.wgsl](../assets/shaders/diffuse.wgsl) に**水量そのものの拡散**(簡易 Richards、流束の重み = max(両セル水量)^γ、質量保存・マスク境界 Neumann・ペン線透水率/アクティブタイル対応)を追加して両方を廃止した(設計判断・A/B の観点は [note/07](note/07-capillary-water.md))。あわせて transfer の吸着を (2−w)×(1−0.7w) の2重カーブから `deposit_weight(w)` の1本に統合(当初は旧同値の単調減少 → 目視「沈着が速すぎる」を受けて**しきい値型** `mix(2.0, DEPOSIT_WET_FLOOR, smoothstep(0, 0.35, w))`=水膜が切れるまで沈まない、へ変更。目視2回目で床 0.3→0.1)。目視3回目「既存色が新しい塗りへ戻らない」で `lift_rate` 既定 0.02→0.1(既存色の復路は再浮遊→拡散だけのため。note/07)。目視4回目「乾き際の紙のざらざら感」で velocity.wgsl に `FLOW_MIN`=0.3(薄い水膜の流れ止め=潤滑近似。しきい値型吸着で低水量まで浮遊するようになった顔料が、紙目駆動の流れで谷へ寄るのを防ぐ)。続けてユーザー判断「**紙の質感再現は要らない**」により `paper_amp`/`paper_gran`/`paper_wet` の既定を全て 0 に(機構・紙ハイトテクスチャ・スライダーは残置=戻すのは値を上げるだけ。副作用: 顔料個性 γ(粒状感)は paper_gran=0 だとほぼ不活性。鉛筆の粒状感 pencil_gran は対象外)。しばらく使って確定なら、紙ハイト機構ごと削る掃除は別途(R 項目相当)。`paint_spread` は削除(広がる距離は brush_water×brush_charge という物理から出る)し、塗るツールのノブは**筆の含み・下の色を溶かす**の2本に。feed splat は目標水位への max 補充+CHARGE_PIGMENT の顔料のみ。新定数: `WATER_DIFFUSE`=0.15 / `WATER_GAMMA`=2.0(diffuse.wgsl)・`DEPOSIT_WET_FLOOR`=0.1 / `DEPOSIT_DRY_POINT`=0.35(transfer.wgsl)。`SOAK_RADIUS`・`WET_HOLD` は消滅。SimParams 224B 不変(`_pad_paint: [f32;2]`)。検証(cargo test --workspace 全32件 / clippy --workspace 警告0 / naga=別 target dir)+実機の目視調整4ラウンド(上記)。採否の判断は plan.md §4 の先送り表(流体ソルバ・紙の質感の行)に記録。残る目視観点(含みでの保持広がり・白紙輪郭の不変・「押しのける」表情の要否)は通常使用の中で確認し、距離が足りなければ WATER_DIFFUSE/WATER_GAMMA をホットリロードで増強、表情不足なら弱い放射流のハイブリッド(note/07 §5)。

**KM の設計判断**: 層「内」の混色は M1c の mixbox のまま(RGB 3ch の K,S で混ぜると黄+青が濁り、mixbox に移した意味が消える)。**KM は層「間」の光学スタックだけに使う**。各層の反射率 R・透過率 T は、その層を白地・黒地に置いた mixbox 発色 R_w, R_b から導く(単層を白/黒地に置いた合成式を逆に解くと `R=R_b`, `T²=(R_w−R_b)(1−R_b)` と閉じた形になり sinh/cosh 不要 = オーバーフロー対策も不要)。この簡約が Curtis 一般式と一致することを km.rs のテスト(reduction_matches_general)で担保。**「顔料ごとの K,S プリセット」は不採用**とし、代わりに白/黒地発色から層単位で導出する方式にした(mixbox 隔離 = plan §4 を保ち、混色モデルを二重化しないため)。GPU 配管は黒 latent を pigment.rs の latent_uniform に追加([12],[13])しただけ。パラメータ追加: `compose_mode`(既定 1=KM。SimParams 末尾で 16B 境界を担う)。

- WGSL 共通定義の連結ロード: SimParams / Splat の struct 定義を [assets/shaders/common.wgsl](../assets/shaders/common.wgsl) に1箇所化し、Rust 側で各シェーダーの先頭に連結してコンパイル。「パラメータ追加 = WGSL 1行」をシェーダーが増えても維持するため
- WGSL コンパイル可能性テスト: 実行時ロードのため cargo build では壊れた WGSL を検出できない。naga(wgpu と同バージョン)でパース+検証する [tests/shader_compile.rs](../tests/shader_compile.rs) を追加(挙動はテストしない方針のまま、コンパイル可否だけ守る)
