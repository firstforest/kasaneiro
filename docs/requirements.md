# 要件仕様

かさねいろ(kasaneiro)が「何を実現するツールか」をまとめる。実装の現在地は [status.md](status.md) が正典、実装計画・順序は [plan.md](plan.md)、技術調査は [note/00-overview.md](note/00-overview.md) 起点の5ノートを参照。

最終更新: 2026-07-05(M3 完了・M4 進行中の時点)

## 1. プロジェクトの目的

Rust + wgpu 製の**水彩シミュレーションペイントツール**(Windows 個人用)。市販ツールでは得にくい「水彩らしい気持ちよさ」を、流体シミュレーション+物理ベース混色で実現する。配布・商用化は当面しない(個人の道具)。

## 2. コア要件(3つ)

### 要件1: wet-on-wet — 色が馴染むグラデーション

先に置いた色の近くに別の色で描くと、**馴染んで綺麗なグラデーションになる**。

- 水の流体シミュレーション(Curtis 1997 簡略版)で顔料が水の流れに乗って移流・拡散する
- 混色は物理ベース(mixbox)。**黄+青を隣接させると濁らず緑に馴染む**のが判定基準
- にじみはストローク領域(濡れ領域マスク)の外へは広がらない — 乾いた紙との境界は壁
- 紙の凹凸(紙ハイト)がにじみの前線・ストリーク・粒状感に表情を与える

### 要件2: グレージング — 乾いた色の上に綺麗に重なる

乾いた色の上に別の色を描くと、**下の層が動かず、重なりが透けて見える**。

- 湿レイヤーは常に1枚。「乾かす」ボタンで乾燥レイヤー(顔料濃度のまま)に焼き込む
- レイヤー合成は Kubelka-Munk の R/T 光学合成(multiply と切替可能)。薄い層ほど下が透ける「内側から光る」グレーズ表現
- 乾いたレイヤーの再編集はしない(焼き込みは一方通行)。いじり続けたい層は Fast Dry(水だけ除去)で止めて焼かない運用

### 要件3: 削り — リフティングと完全消去の2ツール構成

- **リフト(削り)**: 再湿潤して沈着顔料を浮遊層へ戻す。ステイニングの強い顔料(ω 高)は紙に色が残り(ステイン床)、紙の凸部から先に剥がれる
- **消去**: 湿レイヤーの水・顔料・濡れマスクを機械的にゼロへ(紙の白まで戻す。Painter 式)

## 3. ツール構成

| tool | 名称 | 挙動 |
|---|---|---|
| 0 | 描画 | 水+初速+顔料を置く |
| 1 | リフト(削り) | 再湿潤して沈着顔料を浮遊層へ戻す(ω と紙凸で変調) |
| 2 | 消去 | 水・顔料・濡れマスクをゼロへ(完全消去) |
| 3 | 水筆 | 水を置きつつブラシ下の顔料を近傍平均へ均す(半径2固定の局所ならし。なでても濃くならない) |
| 4 | ならし | 水筆と同じ質量保存の箱ぼかしをブラシスケールの広い近傍で行う(濃い山を周囲へ均一に伸ばす) |
| 5 | 鉛筆(下書き) | 下書き線画レイヤーにグレーの粒状線を描く(紙ハイトで濃度変調、筆圧→濃さ)。境界効果なし |
| 6 | ペン(清書) | 清書線画レイヤーに濃色のくっきりした線を描く(筆圧→太さ)。**水の境界になる** |
| 7 | ハイライト | 最上段の白レイヤーに不透明で塗る(筆圧→不透明度) |

ツール 5〜7 にはそれぞれ**消しゴムモード**(トグルで同じブラシが消す側に反転)を備える。

## 4. 線画とハイライト(M4.5)

下書き → 清書 → 彩色 → ハイライトのワークフローを支える追加機能。**ラスタ実装**(専用テクスチャ。ベクター化は先送り — plan.md §4)。

### 線画 — 下書き(鉛筆)と清書(ペン)

- **鉛筆(下書き)**: グレーの粒状線。黒鉛が紙の凸に乗る質感を紙ハイトの濃度変調で出す。アタリ専用で塗りには干渉しない
- **ペン(清書)**: マジックのような濃色のスムーズな線
- 下書き・清書は独立に表示切替できる(下書きだけ消して仕上げられる)
- **合成位置は固定**: 紙 → 乾燥レイヤー → 湿レイヤー → 線画(下書き・清書) → ハイライト。水彩は透明なので線画が上でも塗りは透けて見える

### 線画の境界効果(はみ出し防止)

- **清書ペンの線は水の「土手」になる**: 拡散・にじみ拡張・流れがペン線を越えにくくなる(透水率方式 — 線の濃度に応じて流束を減衰)
- ブラシで線を跨いで直接描いた分はそのまま置かれる = **明示的な筆使いなら境界を越えられる**
- 境界の強さはスライダーで調整可能(0 で従来どおり)。下書き鉛筆は境界にならない

### ハイライトレイヤー

- 常に最上段の白レイヤー。**不透明ブラシ(流体シミュなし)**の白ガッシュ風で、下の層は一切動かない
- 消しゴムモード・表示切替あり

### 線画・ハイライトの Undo/Redo

- 鉛筆・ペン・ハイライトのストローク(消しゴム含む)は**多段 Undo/Redo** できる(Ctrl+Z / Ctrl+Shift+Z)
- 方式はストローク履歴の決定論的再ラスタライズ(流体を通らないので厳密に再現できる)。ブラシ設定を変えた後に Undo/Redo しても過去の線の形は変わらない
- 湿レイヤー(水彩)は対象外 — こちらは undo 1段(M6、plan.md §4)

## 5. 入力要件

- **筆圧対応ペン**(Windows Ink / WM_POINTER)。経路は egui 標準の Touch イベントで完結(octotablet はデッドロックバグで不採用 — status.md M1.5)
- 筆圧 → 半径・水量・顔料量へのマッピングは γ カーブ付きでスライダー調整可能
- マウスでも描ける(筆圧 1.0 固定)。ペン接地中はマウス入力を無視(パーム・二重ストローク対策)
- WinTab 非対応、傾き・ホバー距離は使わない(先送り — plan.md §4)

## 6. UI 要件

- **「乾かす」ボタンは常時見える位置**(スクロール外の左パネル最上部。Fresco の UX 教訓)。あわせて Fast Dry(水だけ除去、焼き込まない)/ Wet the Layer(全面再湿潤)の計3ボタン
- **レイヤーパネルはキャンバスの右**に置き、合成順(上=手前: ハイライト → 清書ペン → 下書き鉛筆 → 水彩(湿)→ 乾燥 → 紙)で1列に並べる。可視性切替+並べ替え(KM 合成で順序が効く。線画3枚は並べ替え対象外・位置固定)
- **レイヤー選択がツール系統の切替**: レイヤーごとに使うツールが決まっているため、選択中のレイヤーを常にハイライトし、左パネルにはそのレイヤーに関係するツール・スライダーだけを出す(水彩=水ブラシ+パレット / 線画=各線画ツール / 乾燥=編集不可の案内+描画ブロック)
- 顔料セレクタ(4スロットの色見本ボタン、ホバーで ρ/ω/γ 表示)
- 全シミュレーションパラメータのスライダーパネル(リアルタイム反映)
- 日本語 UI(Windows システムフォントをフォールバック登録)

## 7. 試行錯誤ループの要件(H1〜H6)

品質はアルゴリズム選択ではなく**パラメータ調整の反復回数**で決まる、が本プロジェクトの核心原則。調整ループ1周を数秒以内に収める6つの装備を維持する(全て実装済み):

| # | 装備 | 要件 |
|---|---|---|
| H1 | WGSL ホットリロード | シェーダー保存で即再コンパイル。エラー時はクラッシュせずオーバーレイ表示+直前の正常パイプラインで続行 |
| H2 | パラメータパネル | 全パラメータを `SimParams` に集約。**追加 = フィールド1行+スライダー1行+WGSL 1行**の定型作業を維持 |
| H3 | プリセット保存/読込 | SimParams ⇄ JSON(`assets/presets/`、git 管理)。パラメータが増えても古い JSON が読める(serde default) |
| H4 | デバッグ表示切替 | 通常 / 水量 / 速度場 / 湿りオーバーレイ / 浮遊顔料 / 沈着顔料 / 紙ハイト の7モード |
| H5 | ストローク記録・再生 | 補間前の生ポインタ入力を記録(`assets/strokes/`、git 管理)。再生時にパラメータを変えて同一ストロークで A/B 比較できる |
| H6 | シミュレーション制御 | 一時停止 / 1ステップ実行 / 速度倍率 / リセット / PNG スナップショット(プリセット名+タイムスタンプ) |

## 8. 非機能要件

- キャンバス 512²(M8 で 512/1024/2048 選択式へ)で 60fps。dev ビルドでも成立させる(`[profile.dev.package."*"] opt-level = 2`)
- マイルストーンの完了条件は数値ではなく**目で見て判定できる体験**で定義する
- `km.rs`(KM 純関数)と mixbox 混色は CPU 参照実装+`cargo test`。流体シェーダー本体はテストせずデバッグ表示で診断。WGSL は naga によるコンパイル可能性テストのみ守る
