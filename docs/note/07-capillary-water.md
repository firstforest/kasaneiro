# 水の毛細管拡散 — 置き馴染みパッチ群の一次原理化(試作)

2026-07-09、note/06 の一連の調整の直後に始めた設計刷新の記録。動機はユーザーの2つの指摘:

1. 「そもそものアルゴリズムは無駄に複雑になってないでしょうか?」
2. 「パラメーターが増えると今後の拡張が難しくなるので、今の時点で一度改善を検討したい」

ブランチ `refactor/capillary-water` 上の**試作**。目視での A/B(main との比較)を通るまで main へはマージしない。

関連ノート: [06-wet-blending-tuning.md](06-wet-blending-tuning.md)(置き換え対象のパッチ群がなぜ生まれたか)/ [01-fluid-simulation.md](01-fluid-simulation.md)(土台の Curtis 1997)

## 1. 診断: パッチ3つは同じ1つの欠落の代償だった

note/06 の5つの仕組みを見直すと、性格が2群に分かれる:

- **筋がいいもの**: 筆の含み(入力モデルの穴の補完。Curtis 本家も筆のリザーバーを持つ)/ 混ざりの水依存 γ(物理的に素直な非線形化)/ 色の溶かし戻し(リフトのレシピの再利用)
- **現象論パッチ**: 置き馴染み=水の足場(水を運ばず**テレポート**させる)/ 広がる勢い=外向き放射流(**relax パスと綱引き**する強制項。feed で毎フレームかけ直してようやく効く)

後者2つが埋めていた欠落は同一で、**このモデルには毛細管現象(水自身が濡れた紙を伝って広がる力)がない**こと。velocity.wgsl の水の移動手段は「水面勾配→加速→移流」だけなので、置いた水の山は平らになると止まり、relax が外向きの発散を消す。

さらに transfer の吸着は `(2−w)`(元からのヒューリスティック)× `(1−0.7w)`(水持ち)と、1つの概念(水量→吸着)に2重のカーブを持っていた。

## 2. 対策: 水高さの拡散1本(簡易 Richards)

[diffuse.wgsl](../../assets/shaders/diffuse.wgsl) の反復ループで、顔料と同時に**水量そのものを拡散**させる:

```
wflux += cap_weight(w_self, w_neighbor) × perm × (w_neighbor − w_self)   // 濡れセル間のみ
w' = w + min(WATER_DIFFUSE×dt, 0.2) × wflux
cap_weight(a, b) = max(a, b)^WATER_GAMMA
```

- **max^γ 重み**(顔料の平均^γ と違う)にした理由: 毛細管の吸引は受け側が乾いているほど強い。平均だと「たっぷりの筆 × 乾きかけの下地」の界面で流束が死に、round 1 の問題(乾きかけに置いても広がらない)が再発する。max なら濡れ側の水量で流束が立つ
- **水量が全体に下がると max も下がって流束が消える** = 「水が引くと止まる」が同じ式から出る(γ カーブと同じ思想)
- 対称流束なので**質量保存**。濡れマスク境界は Neumann(白紙の輪郭は広がらない)。ペン線の透水率(M4.5b)・アクティブタイル(M6)は顔料拡散と同じ経路で自然に効く
- 陽解法の安定条件(per-iter ≤ 0.2)も顔料と同じ。速さは diffuse_iters(12)で稼ぐ

### 何がこれで不要になったか

| 旧機構 | 置き換え |
|---|---|
| 水の足場(paint_soak / SOAK_RADIUS) | 筆が置いた水が毛細管拡散で**実際に運ばれて**足場になる(テレポート廃止) |
| 外向き放射流(paint_spread) | 水の拡散が外向きの水位差を作り、顔料は γ 重み拡散+勾配流で後追いする(relax との綱引き廃止) |
| 吸着の2重カーブ((2−w)×(1−0.7w)) | 単一カーブ `deposit_weight(w)`([transfer.wgsl](../../assets/shaders/transfer.wgsl))。当初は旧カーブとほぼ同値の `mix(2.0, 0.3, w^0.7)` にしたが、初回の目視で「沈着が速すぎる」(毛細管拡散は水の山を薄く延ばして運ぶため、中間水量帯の吸着で色が移動の途中で固まる)と分かり、**しきい値型** `mix(2.0, DEPOSIT_WET_FLOOR, smoothstep(0, DEPOSIT_DRY_POINT=0.35, w))` に変更 = 水膜が切れるまで沈まない。目視2回目「まだ速い」で床を 0.3→0.1 に(毛細管拡散は移流より運びが遅いぶん、浮遊時間を長く取る必要がある) |
| feed の「広がる勢いのかけ直し」 | feed は目標水位への max 補充+少量顔料だけ。広げるのは毛細管拡散の仕事 |

筆の含み(brush_charge)・溶かし戻し(paint_pickup)・γ 重み(DIFFUSE_GAMMA)はそのまま。

目視4回目「乾いてきた時の紙のざらざら感が気に食わない」を受けて [velocity.wgsl](../../assets/shaders/velocity.wgsl) に **`FLOW_MIN`=0.3(薄い水膜の流れ止め、潤滑近似)** を追加。原因: しきい値型吸着で顔料が低水量まで浮遊し続けるようになった結果、乾きかけの薄い水の紙目(paper_amp)駆動の流れが浮遊顔料を谷へ寄せ集め、粒状感 γ がほぼ 0 の顔料でもざらついた(低水量では顔料拡散の重み avg^4 も死んでいるため、寄った顔料を均す機構がなかった)。実物でも薄膜の流量は厚さの3乗で消える(潤滑近似)ので、`vel ×= smoothstep(0, FLOW_MIN, w)` で乾きかけの流れを止め、「流れが止まる(w<0.3)→定着が仕上がる(w<0.35 から立ち上がり)」の順序にした。紙目ストリーク自体は中〜高水量の間に形成されるので残る。

目視3回目「新しい色は既存の塗りへ広がるが、既存の色が新しい塗りの方へ広がってこない」を受けて `lift_rate` 既定 0.02→**0.1**。新しい色は浮遊層(動ける)・既存の色は沈着層(動けない)という非対称があり、水の山からの移流は外向きなので、既存色が新ストロークへ戻る経路は**再浮遊(lift_rate×w×(1−ω))→濃度差拡散**だけ。ここを速くすると新しい水が乗った既存色がほどけて戻る。ステイニング ω の「剥がれない」個性はそのまま効く(フタロ/キナクリドンはほぼ戻らない=仕様)。

## 3. パラメータへの影響

- **`paint_spread` を削除**。「広がる距離」はパラメータではなく物理から出る: 置く水の量(`brush_water`)×含みの長さ(`brush_charge`)が多いほど遠くへ広がる — アナログ水彩の直感と同型
- 塗るツールのスライダーは**筆の含み・下の色を溶かす**の2本に
- 新しい定数(いずれも WGSL、ホットリロードで再調整可): `WATER_DIFFUSE`=0.15 / `WATER_GAMMA`=2.0(diffuse.wgsl)、`DEPOSIT_WET_FLOOR`=0.3 / `DEPOSIT_GAMMA`=0.7(transfer.wgsl)。`SOAK_RADIUS` と `WET_HOLD` は消滅
- SimParams は 224B のまま(paint_spread の枠は `_pad_paint: [f32; 2]` へ)

## 4. 予想される挙動差(A/B の観点)

良くなるはず:

- 水の輪(顔料より水が先行するクロマトグラフィー的な前線)が出る — 実物の水彩に近い
- 放射流由来のリング化(置いた中心が薄く抜ける副作用、note/06 §6)が消える
- 水筆で置いた水たまり・全面再湿潤の水も同じ物理で振る舞う(仕組みの特例が減る)

悪くなるかもしれない(要目視):

- **広がる距離が √t に戻る**: 放射流(移流=距離∝t)より届く距離が短い可能性。対策は WATER_DIFFUSE / WATER_GAMMA / diffuse_iters の増強(ホットリロードで試す)
- **押しのける感じの喪失**: 放射流は既存の顔料を外へ押した。拡散は混ぜるだけなので、「筆の色が下の色を押し広げる」表情は弱くなる
- 全体の乾きの体感が変わる: 水が自分で平らになるため、局所の水たまりが早く馴染む(エッジの出方・粒状化のタイミングに影響しうる)

**判定基準**(note/06 の4ラウンドの要望がそのまま満たせているか):

1. 乾きかけの塗りに筆を置く → 色がブラシ半径の数倍へ広がって馴染む
2. たっぷり濡れた場所では遠く自由に、乾きかけでは控えめに
3. 置いたまま保持でなぞらなくても広がる(含み)/ 上げて置き直すと含み直す
4. 水がたっぷり=傾斜がなくても混ざる / 引くと止まって縁が残る
5. 白紙へのストローク輪郭・にじみが従来と変わらない

## 5. ダメだった場合の退路

- ブランチを捨てれば main(3ノブ+定数の note/06 §8 状態)に戻るだけ
- 「距離が足りない」だけなら、毛細管拡散は残して放射流を**弱く**復活させるハイブリッドもある(その場合も足場テレポートは不要のはず)
