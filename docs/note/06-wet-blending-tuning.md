# 筆置きの広がり・馴染みの設計ノート(置き馴染み一式)

「既に描いてある部分に筆を置いたら、綺麗に筆の色が広がるようにしたい」という要望から始まった、2026-07-09 の一連の試行錯誤の記録。**要望 → 診断 → 対策**を4ラウンド繰り返して、水の量を軸にした5つの仕組みに落ち着いた。後から効きを調整したくなったとき・似た挙動を足したくなったときに読み返す。

01〜05 の技術調査ノートと違い、これは**実装後の設計記録**。パラメータの現在の既定値・範囲は [../parameters.md](../parameters.md)(§1 ブラシ・§4 顔料の輸送)が正典。

関連ノート: [01-fluid-simulation.md](01-fluid-simulation.md)(Curtis 1997 の流体モデル。本ノートの仕組みは全てこの上に乗る)

---

## 0. 要望と対応の全体像

| ラウンド | ユーザーの言葉(要約) | 診断 | 対策 |
|---|---|---|---|
| 1 | 既に描いてある部分に筆を置いたら、綺麗に色が広がってほしい | 乾きかけの塗りでは周囲に水がなく、拡散・移流が働かない。既存の色は沈着済みで動かない | **置き馴染み**(水の足場)+**色の溶かし戻し** |
| 2 | もっと遠くまで。水がたっぷりなら自由に広がって混ざるイメージ | 拡散は √t でしか進まない。水の山の勾配流は山が平らになると死ぬ。吸着が水中でも効くので移動中に固まる | **広がる勢い**(外向きの流れ)+**水持ち**(水中で沈まない) |
| 3 | なぞれば広がるのは良いが、色を置きすぎる。筆に色水がたっぷり含まれている時だけ広がるイメージ | 水と押し出しの注入は splat 発生時(=ポインタが動いた時)だけ。置いたまま=何も起きない | **筆の含み**(置くだけで色水が流れ出続ける) |
| 4 | もっと自然に水を伝って広がってほしい。水が多いときは傾斜がなくても混ざり、少なくなると混ざらなくなる挙動に | 拡散の水依存が線形で弱く、たっぷりでも自由に混ざらず、乾きかけでもじわじわ混ざる | **混ざりの水依存**(流束の重み = 水量^γ)+拡散既定の増強 |

共通の設計思想: **「水がどれだけあるか」を全ての効きの軸にする**。たっぷり=自由に動いて混ざる/引いてきた=止まって定着、が各機構で一貫していれば、ユーザーの直感(アナログ水彩の経験)と一致する。

## 1. 置き馴染み(paint_soak / paint_soak_radius)

**問題**: 塗った部分は蒸発(evap_rate)で水が減り、顔料は沈着層に定着する。そこへ塗る筆(tool=0)を置いても水はブラシ円内にしか足されず、周囲の水が少ない → 拡散の重みも移流もほぼゼロ → 色が硬いダマになる。

**対策**([splat.wgsl](../../assets/shaders/splat.wgsl) 描画ブランチ): **このストローク前から濡れマスクが立っている(=既に描いてある)セルに限り**、ブラシ周囲(`paint_soak_radius` × 実効半径)まで水を目標水位 `brush_water × paint_soak` へ持ち上げる。色が広がるための「水の足場」。

設計判断:

- **max 方式**(足すのではなく目標水位まで上げる)— なでても水が積み上がらない。水筆(M4)と同じ流儀
- **ゲートは `was_wet`**(splat ループ前に捕捉した濡れマスク)— 乾いた白紙にはマスクも水も広げないので、**白紙へのストローク輪郭は従来どおり**。マスク外に水を置くと不可視の水が残留する(velocity/transfer が非 wet セルを素通しするため蒸発すらしない)という実装上の理由もある
- 濡れマスクが立っているタイルは tilescan が常に有効化するので、アクティブタイル(M6)と自然に整合する

## 2. 色の溶かし戻し(paint_pickup)

**問題**: 既存の色は沈着層にいて動かないので、新しい色が上に乗るだけで「馴染む」感じにならない。

**対策**: 筆が触れた沈着顔料を浮遊層へ戻す(リフト tool=1 の弱い版)。`paint_pickup × coverage × 筆圧 × (1−ω)` の割合で dep → susp へ転送。筆の色と下の色が**同じ浮遊層**で混ざるので境界が濁らず馴染む。ステイニング顔料(ω 大)は剥がれず残る — 顔料個性(M3)との整合。

注意: feed splat(§4)経由では毎フレーム適用になるため **0.1 倍に減衰**している(そのままだと1秒で筆の下が全部剥がれる)。

## 3. 広がる勢い(paint_spread)+ 水持ち(wet_hold)

**問題**: 足場があっても遠くまで届かない。理由は2つ:

1. 拡散はランダムウォークなので到達距離 ∝ √t(1秒で数テクセル)。遠くへ運べるのは移流(速度上限 vel_max=1 テクセル/ステップ ≒ 60 テクセル/秒)だが、置いた水の山の勾配流は山が平らになると消える。さらに**外向きの放射流は純粋な発散なので relax パス(発散緩和)が積極的に殺す**
2. 吸着の係数 (2−w) は w=1(たっぷり)でも 1 残る → 広がる途中で紙に固定される

**対策1(paint_spread)**: 置いた点から外向きの放射流を速度場へ直接与える。`vel += 方向 × paint_spread × 筆圧 × 減衰リング × clamp(水量, 0, 1)`。**そのセルの水量を掛ける**ことで「水がたっぷりの場所ほど遠く自由に流れる」が式に入る。was_wet ゲートで白紙には効かない。vel_max クランプが安全弁。relax に殺される問題は §4 の「毎フレームかけ直し」が答え。

**対策2(wet_hold)**([transfer.wgsl](../../assets/shaders/transfer.wgsl)): 吸着率に `(1 − wet_hold × w)` を掛ける。水が多いセルでは顔料が沈まず浮遊し続け、沈着は水が引いてから始まる。**これは塗る筆に限らないシミュ全体の性質変更**(0 で従来どおり)。移動中に固まらない+浮遊同士なので混色(mixbox)も進む。

## 4. 筆の含み(brush_charge / charge_pigment)

**問題**: 「なぞり続けないと広がらない」の根本原因 — 水も押し出しも **splat 発生時にしか注入されず、splat はポインタが動いた時にしか生まれない**。置いたまま=何も起きない。なぞる=注入し続ける、だった。

**対策**(CPU: [src/app/mod.rs](../../src/app/mod.rs) `feed_charge` / GPU: splat.wgsl の feed 分岐):

- 筆を下ろすと含み(`brush_charge` 秒)が満たされ、**置いたまま動かないフレームでも CPU が毎フレーム feed splat(`Splat.feed=1`)を1つ積む**
- feed splat は一括注入(+=)を繰り返すと溢れるので中身が違う: 水は置き馴染みの max 補充に任せ、顔料は `charge_pigment`(通常比)だけ、**広がる勢いを毎フレームかけ直す**(←これが本命。放射流は relax と減衰で消えるので、掛け直して初めて遠くまで運ばれる)
- 含みは置いている時間で減り、動いているフレームは通常注入があるので消費しない。筆を上げて置き直すと含み直す(色水をたっぷり含み直すイメージ)
- **顔料をほとんど足さずに「置くだけで広がる」が成立**する(流れ出るのは主に水)

**既知の制約**: ストローク記録(H5)は生ポインタ点のみ記録するため、「置いたまま保持」は再生で再現されない。

## 5. 混ざりの水依存(diffuse_gamma)+ 拡散既定の増強

**問題**: 拡散([diffuse.wgsl](../../assets/shaders/diffuse.wgsl))の流束の重みが両セルの水量平均に**線形**で、しかも既定が弱い(0.15×4反復)。水がたっぷりでも自由に混ざらず、逆に乾きかけでもじわじわ混ざり続ける。

**対策**: 重みを `(水量平均)^γ`(`wet_weight` ヘルパー、`diffuse_gamma`)へ。γ=1 で旧挙動、γ=4(既定)だと w=1.0 で全開・w=0.7 で約1/4・w=0.5 で約1/16。あわせて `pigment_diffuse` 0.15→0.4(WGSL 安定上限 0.2/反復に張り付き)・`diffuse_iters` 4→12 に増強。水量 1.0 のセル間の実効混ざり速度は旧比約8倍、水量 0.3 では γ 乗の減衰で旧同等以下。

- 「水がたっぷり=**傾斜がなくても濃度差だけで**自由に混ざる/水が引く=急に混ざらなくなり縁の形が残る」が1本のカーブで出る
- 拡散は濃度差駆動なので「顔料のない水へ」は差が最大=一番速く広がる(『顔料がなければかなり広がってよい』を自然に満たす)
- 反復3倍のコスト増はアクティブタイル(M6)が濡れ面積に閉じ込める
- 陽解法の安定条件(1反復の係数 ≤ 0.2)は変えていない — 速さは反復回数で稼ぐ(diffuse.wgsl 冒頭コメントの方針どおり)

## 6. 調整ガイド(どのノブがどの体感に効くか)

スライダーの場所: 塗るツール選択時のパネル(置き馴染み・広がる勢い・下の色を溶かす・筆の含み)/ 開発モード → 味付け(広がる範囲・水持ち・含みから出る顔料・混ざりの水依存 γ・拡散率・拡散反復)。

| 体感を変えたい | 触るノブ |
|---|---|
| 広がる**距離** | 広がる勢い(paint_spread)・広がる範囲(paint_soak_radius)・筆の含み(brush_charge) |
| 広がる**速さ・自由さ** | 拡散率×反復(pigment_diffuse / diffuse_iters)・水持ち(wet_hold) |
| たっぷり/乾きかけの**メリハリ** | 混ざりの水依存 γ(diffuse_gamma)。上げるほど「たっぷりの時だけ」が極端に |
| 下の色との**馴染み** | 下の色を溶かす(paint_pickup)・水持ち(wet_hold) |
| 置くだけで出る**色の濃さ** | 含みから出る顔料(charge_pigment)。0 で水だけ |
| **従来に戻す** | 各ノブ 0(γ は 1)。プリセット H3 に封じ込めて A/B するのが速い |

副作用の把握:

- **wet_hold と diffuse_gamma は塗る筆専用ではなく全体の性質**。白紙への単ストロークも内部の均一化が速くなる(マスクは広がらないので輪郭は不変)
- paint_spread を上げすぎると置いた中心が薄くなりリング状になる(実物のバックランに近い見た目。嫌なら中心の顔料保持を足す改修が次の一手)
- feed 中の pickup 0.1 倍・γ の WGSL クランプ(0.01)・soak_radius のクランプ(1〜8)は splat.wgsl / diffuse.wgsl 側にハードコード
